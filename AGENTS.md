# AI Software House

## Engineering Principles

Todos os agentes devem atuar como engenheiros seniores.

### Obrigatório

- Clean Code
- SOLID
- Clean Architecture
- Arquitetura Hexagonal
- Domain Driven Design
- Design Patterns
- Testes automatizados
- Segurança
- Escalabilidade
- Performance
- Observabilidade

## Referência oficial de patterns

https://refactoring.guru/pt-br/design-patterns

## Regras Globais

- Nunca criar código improvisado
- Nunca duplicar lógica
- Nunca quebrar arquitetura
- Nunca ignorar testes
- Nunca ignorar lint/build
- Nunca ignorar documentação

## Documentação obrigatória

Sempre atualizar:

- README.md
- TASKS.md
- DECISIONS.md
- ARCHITECTURE.md
- API_CONTRACT.md

## Workflow obrigatório

1. architect planeja
2. coordinator valida
3. backend_dev implementa backend
4. frontend_dev implementa frontend
5. coordinator valida integração
6. testes obrigatórios
7. documentação obrigatória

## Testes obrigatórios

Sempre executar:

- lint
- build
- unit tests
- integration tests

## Backend Rules

- Repository Pattern
- Service Pattern

- Dependency Injection
- Separation of Concerns

## Frontend Rules

- Component-based architecture
- Reusable components
- Feature-based organization
- State isolation
- Accessibility
- Responsive design
